O mapeamento de portas no Docker é o mecanismo que permite expor uma porta do contêiner para que possa ser acessada a partir da máquina hospedeira, e consequentemente, de fora do ambiente Docker. Por padrão, os serviços que rodam dentro de contêineres não são acessíveis diretamente de fora, pois ficam isolados em uma rede própria. Para tornar um serviço acessível, como uma aplicação web ou um banco de dados, é necessário mapear a porta interna do contêiner para uma porta da máquina. Esse mapeamento é feito usando a opção -p no comando docker run ou através da diretiva ports no arquivo docker-compose.yml. A sintaxe segue o formato porta_host:porta_container, onde a primeira representa a porta na máquina física ou virtual onde o Docker está rodando, e a segunda representa a porta interna do serviço no contêiner. Por exemplo, ao mapear 8080:80, qualquer requisição feita para a porta 8080 do host será redirecionada para a porta 80 do contêiner, que é onde normalmente o Nginx ou outro servidor web estaria escutando. Esse mecanismo garante que múltiplos contêineres possam rodar isoladamente, mas ainda assim permitir que alguns serviços estejam acessíveis externamente, conforme a necessidade da aplicação.
